# Kina Design System

## Principles

The Kina interface is designed to support human interaction rather than compete with it.

The UI should feel:

- Calm
- Intentional
- Modern
- Trustworthy
- Minimal

The phone is a facilitator, not the focus of the meetup.

## Foundations

- Semantic color tokens
- Consistent spacing
- Accessible typography
- Subtle motion
- Reusable primitives

## Layout System

Kina uses reusable layout primitives to maintain consistent spacing and alignment.

Core layout components:

- Container
- Section

Principles:

- Avoid duplicated spacing rules.
- Maintain predictable content width.
- Prioritize readability on mobile devices.
--
## UI Primitives

Primitive components provide reusable building blocks.

Current primitives:

- Button
- Card
- Badge

Design principles:

- Components should have simple APIs.
- Native HTML behaviour should be preserved.
- Visual complexity should only be introduced when required.
--
## Typography

Ki uses typography to create a calm and welcoming experience.

Primary font:

Inter

Typography principles:

- Prioritize readability.
- Avoid excessive font variation.
- Use hierarchy through size and spacing.
- Keep interfaces conversational rather than corporate.
